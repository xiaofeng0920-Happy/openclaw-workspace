# bank-statement-analyzer

银行对账单自动分析技能。支持 PDF 格式银行对账单的提取、转换和分析，自动生成财务报告。

## 触发条件

当用户提到以下关键词时激活此技能：
- "银行对账单"
- "对账单分析"
- "财务分析报告"
- "PDF 转 Excel"
- "银行流水分析"
- "收支分析"
- 上传银行对账单 PDF 文件

## 功能

### 1. PDF 转 Excel
- 提取 PDF 对账单中的表格数据
- 生成结构化 Excel 文件
- 支持多个月份批量处理

### 2. 财务分析
- 现金流趋势分析
- 收入/支出分类统计
- 月度收支对比
- 重点事项识别

### 3. 报告生成
- Markdown 格式分析报告
- HTML 格式可视化报告
- Excel 数据汇总

### 4. 风险识别
- 连续大额支出识别
- 异常交易模式检测
- 现金流预警

## 输出文件

| 文件 | 格式 | 说明 |
|------|------|------|
| 银行对账单_YYYY.MM-YYYY.MM.xlsx | Excel | 完整数据汇总 |
| 财务分析报告_YYYY.MM-YYYY.MM.md | Markdown | 文字版分析报告 |
| 财务分析报告_YYYY.MM-YYYY.MM.html | HTML | 可视化报告 |

## 依赖

- Python 3.8+
- openpyxl
- pdfplumber (或 pdftotext 命令行工具)
- markdown

## 使用示例

### 示例 1：分析单个 PDF
```
用户：帮我分析这个银行对账单 PDF
[上传 PDF 文件]
```

### 示例 2：批量分析
```
用户：把 2024 年 4 月到 2026 年 2 月的对账单都分析一下
```

### 示例 3：生成报告
```
用户：给我做一份完整的财务分析报告
```

## 注意事项

1. PDF 文件需要是标准表格格式
2. 确保 PDF 文本可提取（非扫描件）
3. 大额支出会自动标记供审核
4. 敏感数据建议本地处理
