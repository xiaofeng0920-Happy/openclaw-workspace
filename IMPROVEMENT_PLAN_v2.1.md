## 🔧 实施步骤

### 阶段 1：核心基础设施（P0 - 本周）

#### 1.1 SQLite 索引层
**目标：** 创建 `state.db` 存储会话元数据，实现 FTS5 全文搜索

**实施内容：**
- 创建 `invest_state.py` 模块
- 设计数据库 schema（sessions, messages, holdings, products）
- 实现 FTS5 全文搜索
- 迁移现有 Markdown 数据

**验收标准：**
- [ ] `state.db` 创建成功
- [ ] 支持会话历史查询
- [ ] 支持持仓记录查询
- [ ] 支持产品规模查询
- [ ] FTS5 搜索功能正常

---

#### 1.2 技能版本管理
**目标：** 在 SKILL.md 中增加 version 字段，支持技能迭代追踪

**实施内容：**
- 更新所有 SKILL.md 增加 frontmatter
- 创建 `skills/.version` 索引文件
- 增加技能更新日志机制

**验收标准：**
- [ ] 所有 SKILL.md 包含 version 字段
- [ ] `.version` 索引文件自动生成
- [ ] 支持技能版本查询

---

#### 1.3 统一 cron 配置
**目标：** 创建 `invest.yaml` 集中管理定时任务

**实施内容：**
- 创建 `invest.yaml` 配置文件
- 迁移 HEARTBEAT.md 中的 cron 配置
- 增加任务执行日志
- 增加失败重试机制

**验收标准：**
- [ ] `invest.yaml` 配置完整
- [ ] cron 任务执行日志记录
- [ ] 失败任务自动重试
- [ ] HEARTBEAT.md 引用新配置

### 阶段 2：增强功能（P1 - 下周前半）

#### 2.1 Profile 多账户支持
**目标：** `invest.py --profile` 支持多产品账户隔离

**实施内容：**
- 创建 `profiles/` 目录
- 为每个产品创建独立配置文件
- 修改 `invest.py` 支持 `--profile` 参数
- 隔离持仓数据、规模数据

**验收标准：**
- [ ] `profiles/前锋 1 号.yaml` 等配置文件创建
- [ ] `invest.py --profile 前锋 1 号` 正常工作
- [ ] 不同 profile 数据完全隔离

---

#### 2.2 模型回退链
**目标：** 配置 2-3 个备选模型，自动故障切换

**实施内容：**
- 在 `invest.yaml` 中配置 fallback_providers
- 修改数据获取模块支持模型切换
- 增加模型健康检查

**验收标准：**
- [ ] 主模型故障时自动切换
- [ ] 切换日志记录
- [ ] 支持手动指定模型

---

### 阶段 3：性能优化（P2 - 下周后半）

#### 3.1 子代理并行化
**目标：** 数据收集、分析、报告生成并行执行

**实施内容：**
- 使用 Python asyncio/multiprocessing
- 将数据收集、分析、报告生成分离为独立任务
- 实现任务队列和结果聚合

**验收标准：**
- [ ] 并行执行时间减少 50%+
- [ ] 任务失败不影响其他任务
- [ ] 结果正确聚合
